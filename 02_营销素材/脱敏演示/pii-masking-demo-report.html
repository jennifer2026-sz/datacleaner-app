Command Reference

Three modes, one command — and full recovery for admin mode

🚀 Quick Start — One Command, Three Modes

dc scrub-dump employees.csv --level external    —  Public release, full anonymization
dc scrub-dump employees.csv --level internal    —  Internal teams, partial mask
dc scrub-dump employees.csv --level admin --password mykey  —  CEO access, tokenized & recoverable
dc recover employees_scrubbed.csv --password mykey      —  Restore original data

Key Capabilities

Built for enterprise deployment — verified across 174 automated tests

🔐

AES-256-GCM Recovery

Admin mode uses military-grade encryption. All tokenized data is fully recoverable with the master password via dc recover.

🔗

Deterministic Hashing

SHA-256 ensures the same input always produces the same output. Database JOINs survive scrubbing — two tables with the same email stay connected.

📋

Format Preservation

Phone numbers still look like phone numbers. SSNs retain their structure. BI dashboards keep working because data patterns are maintained.

🔍

Auto-Detection

35 PII column patterns recognized automatically — names, emails, phones, SSNs, credit cards, passports, IPs, and more. No manual column mapping needed.

🏠

100% Local Processing

No data ever leaves your machine. No cloud API calls. Offline license validation. GDPR, HIPAA, and CCPA friendly by design.

⚡

Streaming Architecture

Handles files exceeding 100 MB with chunked streaming — never loads the entire dataset into memory at once.
